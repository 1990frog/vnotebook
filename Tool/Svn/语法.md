# 语法

## SVN服务端：

创建版本库：svnadmin create /path/repos  
删除版本库：rm -rvf /path/repos  

版本库配置及权限分组：  
authz配置用户组以及用户组权限  
passwd配置用户名和密码  
svnserve.conf配置默认权限、权限配置文件及密码配置文件  

启动svn：svnserve -d -r /svnresponse/

配置文件：svnserve.conf(通用配置)
参数：
anon-access = read (没有用户名密码的用户，参数：none、read、write)
auth-access = read (有用户名密码的用户，参数：none、read、write)

password.db 指定passwd文件位置
authz.db 终端authz文件位置

配置文件：passwd(用户文件)
user = password

配置文件：authz(用户文件)
[groups] //用户组
group = user1,user2,user3

[/]
@group1 = rw
@group2 =r 

[imooc:/]
@group1 =rw
user1 = rw

[repos:/xxx]
* = r //*所有用户

常见svn客户端命令
svn add - 添加到版本控制
svn commit(ci) - 提交修改到服务端（创建一个新版本号）
svn update(up) - 更新工作副本（默认情况下，每个文件只会从服务端更新一次最新版本）
svn delete(del) - 从版本库中删除文件或目录

mkdir {css,js,img} linux命令
touch {a.css,b.css}
touch a.css b.css

svn add js --non-recursive 只增加js文件夹，不增加文件夹下文件
svn add * 增加全部文件到svn服务器（不扫描已经增添的文件目录）
svn add * --force 所有目录全部扫描

svn ci -m "this is index" index.html
svn ci -m "" index.html (-m参数必加)

svn up -r 1 index.html 更新为版本1
svn up * 强制更新到最新版本（更新到历史版本，版本号未变，使用up * 覆盖本地版本）
svn up （本地缺失文件，有最新版本）

svn delete

+svn ci -m ""

svn diff(di) 版本差异比较（-、+）
svn mkdir 创建目录并增加到版本控制
svn cat 不检出工作副本直接查看指定文件

svn di index.html
svn di -r 2 index.html 比较工作副本与第二个版本差异
svn di -r 1:3 index.html 比较第一个版本与第三个版本差异
svn di 比较所有文件差异

svn mkdir doc 创建一个名为doc的文件夹直接提交到svn中
svn cat svn://ip/index.html 不检出文件直接查看文件的内容

工作副本还原：svn revert [--recursive][filename|*]
svn revert * 恢复当前目录全部修改过的文件
svn revert --recursive * 递归恢复

多版本库解决方案：
多个端口号：
优点-版本库可以创建在任意位置
缺点-端口号容易混淆
一个端口号：
优点-版本库必须创建在同一个目录下
缺点-无需分配端口号
