方案对比
直连
优点：

+ 简单方便
+ 适用于少量长期连接的场景
缺点：

+ 存在每次新建/关闭TCP开销
+ 资源无法控制，存在连接泄露的可能
+ Jedis对象线程不安全

连接池
优点：

+ Jedis预先生成，降低开销使用
+ 连接池的形式保护和控制资源的使用

缺点：

+ 相对于直连，使用相对麻烦，尤其在资源的管理上需要很多参数来保证，一旦规划不合理也会出现问题。
