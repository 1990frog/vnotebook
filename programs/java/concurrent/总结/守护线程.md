# 线程两大类
守护线程、用户线程

# 作用
给用户线程提供服务

# 有以下两种情况发生程序才能结束
1.调用System.exit()
2.代码执行完毕
当jvm发现所有我们自己创建的用户线程都结束了，它就会和这些守护线程一起结束。因为守护线程的作用就是给其他线程提供服务，被服务的线程都关闭了，守护线程也就没有存在的价值（例如垃圾处理器）。

# 守护线程的3个特性
1. 线程类型默认继承自父线程
2. 被谁启动（jvm自动启动，main函数不是守护线程）
3. 不影响jvm退出

# 守护线程和用户线程的区别
整体无区别
唯一区别在于jvm的离开

# 我们是否需要给线程设置为守护线程？
不应该，一旦设置成守护线程，jvm发现只剩下守护线程，可能会关闭jvm。这有可能会造成数据不一致。
