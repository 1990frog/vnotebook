# newsql
除了sql和NoSQL外还有一种非常流行的数据库存储系统称之为newsql。
newsql系统通常可以使用sql对数据库进行操作也就是说具有sql的特征，另外数据存储上大多使用分布式存储方式，因此可以方便的对存储进行扩充，常见的newsql系统有：tidb和cockroachDB

# 关系型数据库
## 特点
+ 数据结构化存储在二维表中（大部分现在也支持json了）
+ 支持事物的原子性A，一致性C，隔离性I，持久性D特性
+ 支持使用sql语言对存储在其中的数据进行操作

## 场景
+ 数据之间存在着一定关系，需要关联查询数据的场景
+ 需要事务支持的业务场景
+ 需要使用sql语言灵活操作数据的场景

# 非关系数据库
## 特点
+ 存储结构灵活，没有固定的结构
+ 对事务的支持比较弱，但对数据的并发处理性能高
+ 大多不使用sql语言操作数据

## 场景
+ 数据结构不固定的场景
+ 对事物要求不高，但读写并发比较大的场景
+ 对数据的处理操作比较简单的场景

# 关系数据库选型原则
+ 数据库使用的广泛性
+ 数据库的可扩展性
+ 数据库的安全性和稳定性
+ 数据库支持的操作系统
+ 数据库的使用成本
